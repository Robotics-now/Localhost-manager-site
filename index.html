Features

Everything a dev needs.
Nothing they don't.

Built for the kind of developer who has five servers running and can't remember which port is which.

Auto port discovery
Scans all 65,535 ports in parallel batches. Results are cached for up to 24 hours — configurable — so your popup opens in under a second on repeat visits.
Real HTML hosting
Drop an HTML file and it spins up a real http.createServer on a free port via the companion Node.js control server. Not a blob URL — a proper localhost address.
Network scan
Optionally scan devices on your local network (192.168.x.x) across common ports. Discover what's running on your router, NAS, or other machines without leaving the browser.
Smart caching
Auto scan results are stored in extension storage with a configurable TTL (30 min to 24 hr). Inactive ports are pruned automatically. Force a fresh scan any time with Rescan.
Swipe to remove
Drag any port row left to reveal a delete button. Works in both list and grid view. No accidental deletions — swipe right or click away to cancel.
Light, dark & auto theme
Follows your system appearance by default. Switch to forced light or dark in settings. Preference is persisted across sessions.
Live status dots
Green dot = active. Gray dot = known but offline. Blue dot = hosted by the extension. Red pulsing dot = error. At a glance, you always know what's up.
List & grid views
Toggle between a detailed list view (title, URL) and a compact icon grid. View preference is saved and restored per session.
Open API contract
The control server speaks plain JSON over HTTP. Three endpoints. Bring your own server in Python, Go, Rust — anything that can bind a TCP port and respond to POST requests.

Setup

Running in two minutes.

No build step, no package manager for the extension itself. Just load and go.

1
Load the extension
Go to chrome://extensions, enable Developer Mode, click "Load unpacked", and select the extension folder. Note the extension ID shown on the card.
2
Start the control server
The control server is what lets the extension host real HTML files on actual localhost ports. Run it once and leave the terminal open. Requires Node.js — no npm install needed.
node server.js
3
Click the extension icon
The popup scans your known ports immediately. Active services appear with a green dot. Click any row to open it in a new tab. Drag left to remove a port from the list.
4
Optional: enable Auto Scan
Open Settings (gear icon) and toggle Auto Scan on. The extension will probe all 65,535 ports, cache the results, and only re-scan after your chosen interval — down to 30 minutes, up to 24 hours.

API

Build your own server.

The extension talks to the control server over plain HTTP. Implement these three endpoints in any language and it just works.

Base URL: http://127.0.0.1:8765   ·   All responses: Content-Type: application/json   ·   CORS required on all endpoints
GET /status Health check & active port list
Response
"ok": true,
"activePorts": [9000, 9001]
POST /start Serve an HTML file on a given port
Request body
"port": 9000,
"content": "<html>...</html>"
Response · success
"ok": true,
"port": 9000,
"message": "Serving on http://localhost:9000"
Response · failure
"ok": false,
"error": "Could not bind port 9000: address in use"
POST /stop Shut down a hosted server
Request body
"port": 9000
Response · success
"ok": true,
"port": 9000,
"message": "Stopped server on port 9000"
Response · failure
"ok": false,
"error": "No server running on port 9000"
HDR Required CORS headers Must be present on every response 
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type
Preflight
OPTIONS * → 204 No Content

Settings

Configurable to your workflow.

Open settings from the gear icon in the popup. Changes apply immediately — no reload needed.

Theme
Light, dark, or auto (follows system). Segmented control — one tap to switch.
Grid view
Toggle between a detailed list and a compact icon grid. Saved across sessions.
Auto scan
Discover all active ports automatically. Inactive ports are removed. Results cached per your chosen interval.
Scan interval
Slider from 30 min to 24 hr in 1-hour steps. Grayed out until Auto scan is enabled.
Local network scan
Scan 192.168.x.x devices on common ports. Configure the IP base range. Results appear below the localhost list.
Control server status
Settings shows whether the Node.js control server is online and how many ports it's currently hosting.
